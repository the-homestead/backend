# Security Policy

We take the security of this project seriously and appreciate responsible disclosure from the community.

## Supported Versions

Only the latest release on the default branch is actively supported with security updates.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅         |
| Older   | ❌         |

## Reporting a Vulnerability

To report a security issue, please email **[dabz@homestead.systems](mailto:dabz@homestead.systems)** with the following information:

**Subject:** `Security vulnerability report: <short description>`

**Body:**

```
Project name:
Version/commit hash:
Affected component/file:
Type of vulnerability (e.g., RCE, XSS, auth bypass, data leak):
Impact assessment (what an attacker could do):
Steps to reproduce:
Proof of concept (code, screenshots, logs, or URLs if applicable):
Suggested fix or mitigation (optional):
Your name/handle:
How you would like to be credited (optional):
```

## What to Expect

* We will acknowledge your report within **48 hours**.
* We aim to provide an initial assessment within **5 business days**.
* We will work with you to validate the issue and determine a fix.
* Once resolved, we will coordinate a responsible disclosure timeline.

## Responsible Disclosure

Please do not:

* Publicly disclose the issue before we have released a fix.
* Exploit the vulnerability beyond what is necessary to demonstrate it.
* Access or modify data that does not belong to you.

We appreciate your help in keeping this project and its users safe.

## Security Updates

Security fixes will be released as soon as reasonably possible and noted in the project changelog or release notes.

---

Thank you for helping improve the security of this project.
